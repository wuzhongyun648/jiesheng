# 结绳的 toB / toG 商业化框架

口径：把个人科研工具升级成可商业化产品，对齐腾讯 toB / toG 云平台 PM 岗。

核心前提：个人版结绳是入口和演示，不是生意本身；生意是组织级研究记忆层。LLM Wiki 覆盖个人和开发者那一层，结绳的价值在产品化，也就是它没覆盖、而结绳能往上长的部分。

---

## 1. 定位

结绳是跑在腾讯云 / QClaw 上的组织级研究记忆层。它让一个重文献的团队，把读过、判断过、发现的矛盾沉淀成带溯源、可共享、可累积的记忆资产，而不是随人流失、散落各处。

---

## 2. 从个人工具到产品

- 个人层，对应 LLM Wiki 覆盖的范围：一个人的研究记忆。这一层用来证明价值和落地能力，不构成壁垒。
- 产品层，结绳的生意所在：把它做成团队和机构共享、强制溯源、主动报冲突、可本地部署的记忆层。

三个个人版没有、组织版才出现的价值：

1. 跨人。知识不再锁在某个研究员脑里，团队可复用。
2. 抗人员流动。人走了，积累的理解还在。
3. 可信、可审计。每条结论都能回原文核对，高风险决策才敢用 AI。

---

## 3. 客户与痛点

目标客户是重文献、跨时间、强溯源、数据敏感、知识易随人流失的组织：

- toB：企业研发，如药企、生物、硬科技、AI 实验室；投研，如券商研究所、基金；律所；咨询与尽调。
- toG：高校与科研院所、国家实验室、政府研究与智库、政策与情报研究单位。

组织级痛点比个人场景更尖锐：

- 知识碎片化。每个研究员的上下文锁在自己脑里或私人 Zotero，团队无法复用。
- 知识随人流失。人一走，走的是积累的理解，不只是文件。
- 不可信导致不敢用。通用 AI 总结没有溯源，临床、合规、投资、法律等高风险场景不敢用来做决策。
- 现有工具不匹配。NotebookLM、ChatGPT、通用 RAG 不累积、不强制溯源、不报跨文献冲突，对敏感数据也不够本地和安全。

---

## 4. 产品形态

产品是组织级研究记忆层，以腾讯云 / QClaw 的垂直 Agent 能力和 skill 模板交付。关键能力直接来自结绳的差异点：

- 强制溯源。`[原文]` 与 `[推断]` 加字段级校验，是高风险领域敢用的前提。
- 主动跨文献冲突检测。帮团队发现某篇和三个月前的结论矛盾。
- 本地优先、私有部署。研究、临床、法律、政府数据不出域。
- 人在环复核。组织级可控。
- 可累积的记忆。随时间增值为组织资产。

---

## 5. 为什么是腾讯 / QClaw

- 本地优先。QClaw 本地化正好承接数据敏感的 toB / toG，研究、临床、法律、政府数据不能上公有云。这是相对 NotebookLM、ChatGPT 的实际差异。
- skill 与 ClawHub 生态。结绳做成垂直 skill 模板，可复制、可分发；平台卖能力，垂直行业出模板，覆盖科研、医药、投研、法律。
- 腾讯云。提供 toB / toG 的销售通道、企业信任、安全合规和私有化部署能力。
- 微信、QQ 入口。在中国组织内部低摩擦渗透。
- 国产模型，如 DeepSeek、GLM、Kimi。兼顾合规与成本，贴合中国 B 端和 G 端客户。

---

## 6. 商业模式

不与 Notion、NotebookLM 在通用 SaaS 上正面竞争，而是：

- 主推腾讯云 / QClaw 上的组织级研究记忆能力，按席位或按组织授权；数据敏感客户走私有化或本地部署，如政府、药企、金融，做高客单的企业与政府合同。
- 垂直 skill 模板，覆盖科研、医药、投研、法律，扩展 QClaw 的 ClawHub，模板可售、可复用。
- 把记忆层作为基础设施，让其他垂直 Agent 在它之上构建。

主线建议：以腾讯云私有或专属部署的团队研究记忆为主，垂直模板为辅。
