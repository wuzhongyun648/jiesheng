# 结绳（Jiesheng）— OpenClaw / QClaw 技能构建规格

> 这是"结绳"科研记忆搭子的工程规格,用于在 Claude Code 里构建一个 OpenClaw/QClaw 技能（skill）。
> 设计判断已定,**严格按本规格实现,不要自由改动设计**。

---

## 1. 它是什么

结绳是一个 OpenClaw/QClaw 技能。装进 Agent 后,它不再是"问一句答一句"的执行器,而是一个**记得用户整个研究脉络的科研协作者**:记得在做什么课题、读过哪些论文、对每篇有过什么判断;每次回应都站在这个脉络之上,而不是从空白开始。

只做两件事,做透:

1. **持久研究记忆** — 维护一份关于"我的研究"的活档案（课题 / 方法 / 读过的论文及判断 / 待解问题）,作为一切回应的底座。
2. **主动关联提示** — 当新读的论文和记忆里的旧论文产生关系（冲突 / 互补 / 取代 / 可组合）时,**在用户没问的时候主动提示**。

贯穿底线:**强制溯源**（每个结论标来源、可回原文核对）、**区分 `[原文]` 和 `[推断]`**、高风险动作（跑代码 / 改文档）先确认。

---

## 2. 运行目标（runtime）

- 目标平台:**QClaw**（基于 OpenClaw）,同时兼容原生 OpenClaw。
- 技能 = 一个文件夹,含:
  - `SKILL.md` — 自然语言 SOP（教 Agent 何时、怎么做）。
  - `scripts/` — Python 脚本,**只做机械活**（解析 PDF、读写 markdown 文件、后期的索引操作）。
  - `workspace/` — 记忆数据（见第 3 节）。
- **职责划分（重要）**:语义工作（读论文、抽假设、下判断、判断冲突）由 **Agent 按 `SKILL.md` 来做**;脚本只负责机械 I/O。
  - 因此**脚本一般不需要调用大模型,也就不存在 API key 问题**。
  - Agent 用它自带的读 / 写工具直接落盘记忆文件。

---

## 3. 记忆存储:markdown 为唯一真相源（已定）

- **canonical store = markdown 文件**:
  - `workspace/MEMORY.md` — 研究主干（课题 / 方法 / 卡点）。**每轮必加载,要克制,不塞论文细节。**
  - `workspace/memory/papers/{id}.md` — 一篇论文一个文件。这就是"绳上的结",文件越多记忆越厚。
- `workspace/index`（SQLite,**派生缓存,B3 才加**）— 存 `{id, title, 核心假设, 关联, tags}`,用于论文多了之后快速检索 / 过滤;**可随时从 markdown 重建,不是真相源**。
  - MVP 量级（几十～上百篇）先不做,加载全部让模型挑即可。
- **MVP 阶段不上向量库 / 知识图谱**。规模真撑不住时,只换检索后端,**schema 不动**。

---

## 4. 论文文件 schema（`memory/papers/{id}.md`）

字段:

- **出处** — 作者 · 会议 / 年份
- **核心假设** — 每条标 `[原文 §x]`
- **关键结论** — 标 `[原文 §x]`
- **我的判断** — 标 `[推断]`
- **关联** — 指向其他论文 id 的有向边 + 关系类型（冲突 / 互补 / 取代 / 可组合）

标签约定（**字段层面的硬约束,不靠模型自觉**）:

- `[原文 §x]` = 对原文的客观摘录（带章节号）。
- `[推断]` = 我 / Agent 的推断。
- 两者必须视觉上可区分——"可信"落到字段层面,而不是声称自己多准。

示例 `linear-bandit-2011.md`:

```
# Improved Algorithms for Linear Stochastic Bandits
出处: Abbasi-Yadkori et al. · NeurIPS 2011
核心假设: reward 平稳（θ* 固定） [原文 §1]
关键结论: Thm 2 置信椭球构造不依赖平稳性 [原文 §4]
我的判断: regret 界不适用我的非平稳设定；但 confidence set 的构造那段或许能借 [推断]
关联: ⟷ ppr-dynamic-2016 （冲突：平稳 vs 非平稳）
```

## 5. SKILL.md 的 SOP（共 3–4 个技能；MVP 先做"读一篇论文"）

### 技能:读一篇论文

- **触发**:用户丢进一篇 PDF / 问"这篇能用吗"。
- **步骤**:
  1. 结合 `MEMORY.md` 的课题、卡点来读,**不给教科书定义**。
  2. 抽 核心假设 / 关键结论,每条标 `[原文 §x]`。
  3. 写出"我的判断",标 `[推断]`。
  4. ★ 扫描已有论文的"核心假设",逐一比对,**冲突就主动报告**。 ← 主动连点
  5. ★ 存成 `memory/papers/{id}.md`,并回写"关联"边到**两边**文件。 ← 沉淀
  6. 跑代码 / 改文档前,先向用户确认。 ← 高风险底线
- **第 4、5 步是写死的、不可跳过的**（不交给用户自律或模型即兴发挥）。

### 技能:写 related work / 顺脉络检索（B3 再做）

- 写 related work:带出处调出相关论文,产出对比。
- 找下一步:拿"卡点"去检索 arXiv、按贴合度排序、命中接回"待读"。

---

## 6. Demo 要还原的 5 个"啊哈时刻"（验收标准）

1. **它记得** — 丢论文 A,咬住我的设定回答,并带原文出处。
2. **它主动连点** — A 入库后不等我问,指出它与两周前的 B 冲突。
3. **判断留住了** — 写 related work 时,从记忆里直接调出带出处的对比。
4. **它替我想下一步** — 给出有理由的方向、标注边界,并把建议沉淀进记忆。
5. **它带着记忆去查** — 拿卡点检索 arXiv、按课题贴合度排序、命中接回"待读"。

---

## 7. 搭建阶段

- **B0** — 脚手架 + 种子数据（MEMORY.md + 2 篇种子论文）。
- **B1** — "读一篇论文"闭环（SOP 第 1–3 步）→ 命中啊哈 #1、#3。
- **B2** — 主动连点（SOP 第 4–5 步）→ 命中啊哈 #2。
- **B3** — 检索 + 复用（+ 轻量 index + 第二个技能）→ 命中啊哈 #4、#5。
- **B4**（可选）— 接 Mem0 / MemoryOS 做对照,schema 不变。
- **B5** — 录 demo + 复盘。

---

## 8. 约束 / gotchas

- 脚本里**绝不硬编码 API key**;语义工作交给运行时的 Agent / 模型。
- **纯本地文件操作**（QClaw 会扫描可疑网络请求 / 危险系统调用,命中即拦截）。
- **跨平台**:用 `pathlib`,不要 Linux-only 写法（要能在 Windows 的 QClaw 运行时跑）。
- `SKILL.md` 用 OpenClaw 格式（自然语言 SOP + 调用脚本）。
- markdown 是真相源,index 是派生。
